## 初识 Astro

建立本站的初衷是想要一个简洁干净的个人空间。  

在日常编程的过程中，时不时的会发现一些小惊喜，它们可能是迅速提升性能的小众秘法，可能是难以寻找的网络资源，可能是经验总结的血泪忠告，可能是架构方面的奇思妙想，也可能是毫无意义的胡言乱语。  

想要把它们记在某些地方，本地笔记？不能随时查看。平台帖子？想要个性化定制。思来想去个人博客就刚刚好，可以满足随时查看，又没有平台束缚，可以随心所欲，又可以适当分享。  

一开始是想要Nuxt建站，后来经好友推荐，初识Astro。在查看Astro官网的时候被其所描述的性能所震惊，而后又看到了名为Astro Island的局部自定义交互机制，顿时来了兴趣，事实证明确实不错。  

如果你也感兴趣：[点击这里前往官网](https://astro.build/)

## 页面设计

我是不太熟悉前端的，毕竟我绝大多数时间都只处理后端业务，所以页面设计这一块，只能是依靠AI大人了。事实证明，在不熟悉前端这个生态的情况下，我连如何描述问题都没法精准拿捏，没办法，狠狠的去补了一波基础。  

网站源码是部署在Github上的公开仓库，这意味着任何人都可以查看源码。我尽量按照自己理解来稍微整理了下仓库，如果发现写的很差，没错那不是你的错觉，那就是写的很差。  

关于页面整体的设计，倒是没有让AI来决定。首先是大量的直角元素，这是一开始我定下的目标，感觉在生活中的圆角已经泛滥到随处可见，不由得产生了一丝审美疲劳，回头一看，直角配合阴影其实还是很不错的。  

设计灵感的来源多种多样，首先是直角配合阴影是从V社学来的，虽然他们有使用小幅度的圆角，但整体来说还是偏直。其次是Pinterest这个网站，从当年一看见这个网站的第一眼我就喜欢上了这种瀑布流布局，因此博客文章列表我索性直接做成了瀑布流（也算是圆梦了。最后就是页面配色，话不多说直接上焚决：    
[神秘网站](https://storybooks.fluentui.dev/react/?path=/docs/theme-colors--docs)  

而说回页面中间的这个大长方形，一开始其实想的是在瀑布流中间预留出一个空间来放置一些常见功能，例如导航组啊，主题按钮啊，Social Links啊等等，还有最重要的标签系统。我不太喜欢为标签单独开一个页面，所以想尽办法把这些元素全部都塞在一起，然后尽量保持简约。  

那为什么后来又设计出了这个雷霆大长方形呢？  
无它，我不会实现瀑布流镂空这个效果，试了两三遍感觉哪哪不对，索性放弃。但最后其实还是挺满意的，或者说，不赖。  

如果你仔细查看的话会发现一个稍微奇怪的点，那就是启用标签时，标签组会显示在长方形的下方，而且它也会响应滚动。但具体它的滚动行为，额...，我只能说尽力了（其实这也是个意料之外的副产物。

## 部署 Astro

Astro的静态网页部署十分简单，或者说常见的SSG应该都很易用。  

首先当然还是经典的node.js安装，我说一句node.js真王朝了。  

我默认使用的都是@latest，但Astro的安装似乎是有node版本要求的，过低了不行（真有人会用低版本node吗？然后就是一键部署命令：

```bash
> npm create astro@latest
```

在初次调用的时候，它会自动给你把astro的包给打上（如果你没有的话），后续你再次调用该命令的时候就快很多了。  

然后就是选择创建路径，选择模板（出于强迫症我选了Empty），选择Git初始化（如果你选择了的话那么它会自动帮你commit一次，具体的消息是："Initial commit from Astro"。  

一路y之后cd进你的目录，打开你的vsc就可以开始编辑了。（vsc会提示你下载astro的插件）
